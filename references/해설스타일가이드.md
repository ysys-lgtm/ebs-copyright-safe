# 고연승t 해설 스타일 가이드

본 문서는 변형 문제 해설 작성 시 반드시 따라야 하는 고연승t 커리 용어 체계를 정의한다.

## 핵심 용어 8종

| 용어 | 정의 | 해설 표기 | 적용 상황 |
|------|------|---------|----------|
| **god** | 글을 관통하는 필자의 생각/핵심. it-that 강조구문, important, must, key 등 필자의 입장을 드러낸 어휘 | `god 표현`, `god표현인 ~를 통해` | 주제/요지/주장 문제에서 핵심 파악 시 |
| **#Tagging (#tag)** | 반복되는 핵심 요소에 해시태그 표기. keyword 찾기 | `#disclosure`, `#information provision` | 반복 어구/핵심 소재 강조 시 |
| **Piercing (P.)** | 앞 내용을 받아주는 지시어, 대명사, such, the 등 응집성 장치 | `P.they`, `P.these values` | 문장 연결/흐름 분석 시 |
| **Signal (△)** | 연결어(접속사, 연결부사). 글의 흐름 전환 신호 | `△As a result`, `△However` | 논리 전환/순접/역접 분석 시 |
| **CC** | Compare & Contrast. 비교/대조 구조 | `CC구조`, `CC패턴` | 대조되는 대상이 반복될 때 |
| **BS** | 글의 주제와 연관이 없는 포인트 (오답 근거) | `(BS : 이유)` | 오답 선지 분석 시 |
| **TAP** | Tagging + Piercing 합성어 | `TAP 패턴` | 주제/주장/요지 문제 해설 시 |
| **PiSig** | Piercing + Signal 합성어 | `PiSig 패턴` | 순서/삽입 문제 해설 시 |

## 해설 구조: 2단 체계

### [PK POINT] — 정답 논리 해설
- **Penalty Kick / Piercing KICE**: 답이 될 수밖에 없는 핵심 이유를 논리적으로 설명
- **사전적 풀이**: 시중의 흔한 사후적 풀이가 아닌, 독창적 풀이 방식과 용어를 활용
- god 표현 지적 → #태깅 → P./△ 분석 → 정답 도출 논리 순서로 전개
- 톤: ~입니다/~됩니다 강의체

**작성 패턴 예시:**
```
[PK POINT]
첫 문장에서 important라는 god표현을 통해 important advantage of disclosure가 중심 소재라는 것을 알 수 있으므로 #important advantage of disclosure를 태깅해 준다. #disclosure, #information provision이 반복되어 강조되므로 역시 이 모든 정보를 함께 태깅해 준다. ②의 benefits of publicizing information이 이러한 입장을 나타내므로, ② '자유 선택을 보장하기 위한 정보 공개의 이점'이 답이 된다.
```

### [PK GOAL] — 출제 의도 총평
- **평가원의 문제 설계의도**를 간파하며 총평 제시
- 평가원의 트렌드나 논리를 파악해 얻어가야 할 정보 제시
- 한 수 위에서 내려다보는 시야 제공
- 톤: ~입니다/~습니다 강의체, 학습 조언 포함

**작성 패턴 예시:**
```
[PK GOAL]
최근 주제 문항의 정석을 보여주는 지문으로 수능 지문의 간결한 논리를 드러내는 지문입니다. 첫 문장부터 god표현, 즉 'an important advantage of disclosure'를 잡아가는 것이 핵심입니다. 주제 문항의 경우 태깅이 반복되거나, 명확한 god 표현이 나오면 반드시 이에 주목하여 풀어야 합니다.
```

### 오답 분석 — BS 패턴
- 오답 선지 옆에 `(BS : 근거)` 형태로 간결하게 표기
- BS = 글의 주제와 연관이 없는 포인트

**작성 패턴 예시:**
```
① 고객이 공개된 정보를 이용할 수 있도록 하는 단계
③ 기업들이 자유 시장에서 이익을 증대시키기 위한 전략 (BS : 이익을 증대-언급x)
④ 현재의 산업 동향을 파악하고 분석할 필요성 (BS : 산업 동향-지엽적)
⑤ 다양화된 시장이 합리적인 고객 선택에 미치는 영향 (BS : 다양화된 시장-언급x)
```

## 유형별 용어 활용 가이드

| 문제 유형 | 주로 사용하는 용어 | 해설 포커스 |
|----------|-----------------|-----------|
| 주제/요지/주장 | god, #Tagging, TAP | god 표현으로 핵심 파악 → 태깅 반복 확인 |
| 제목 | god, #Tagging | 핵심 소재 태깅 → 포괄하는 제목 매칭 |
| 빈칸 | god, #Tagging, x 타게팅 | 빈칸을 x로 두고 타게팅 → 태깅 → god 파악 → x 결정 |
| 순서 | PiSig, P., △ | 지시어(P.)와 연결어(△)로 단락 연결 |
| 삽입 | PiSig, P., △ | 삽입 문장의 P./△가 가리키는 앞뒤 문맥 |
| 어법/어휘 | god, #Tagging | 정답 근거 + 문법/어휘 포인트 설명 |
| 요약 | god, #Tagging, CC | 핵심 논지의 태깅 → (A)(B) 매칭 |
| 무관한 문장 | #Tagging, BS | 태깅 흐름에서 벗어나는 BS 문장 식별 |

## 빈칸 유형 전용: x 타게팅 해설법

빈칸 문제에서는 반드시 **x 타게팅** 방식으로 해설을 작성한다.

### 원리
- 빈칸 = 미지수 **x**
- "빈칸 x를 ~로 타게팅합니다" → 태깅하며 읽기 → god 파악 → x 결정

### 작성 패턴

**첫 문장** (필수): `빈칸 x를 [빈칸이 묻는 핵심 개념]으로 타게팅합니다.`
**둘째 문장**: `빈칸 문장을 읽으면 '~는 x이다/x를 한다'가 되므로, x가 무엇인지를 찾으며 글을 읽습니다.`

**패턴 A — 빈칸이 문장 내부에 있을 때:**
```
[PK POINT]
빈칸 x를 구매의 본질적 성격으로 타게팅합니다. 빈칸 문장을 읽으면 '구매를 x로 간주해야 한다'가 되므로, x가 어떤 성격인지를 찾으며 글을 읽습니다. #buyers, #risk, #brand, #perception에 태깅하며 글을 읽으면 이 지문의 god는 "구매는 위험 최소화 과정이다"임을 알 수 있다. 따라서 x에 들어갈 말로 가장 적절한 것은 ③이다.
```

**패턴 B — 빈칸이 결론부에 있을 때:**
```
[PK POINT]
빈칸 x를 우연의 일치에 대한 감각으로 타게팅합니다. 빈칸 문장을 읽으면 '그것이 어떤 것을 알게 해 준다'가 되므로, x가 무엇을 알게 해 주는지를 찾으며 글을 읽습니다. #coincidences, #birthday problem, #probability에 태깅하며 글을 읽으면 이 지문의 god는 "우연의 일치는 직관보다 훨씬 자주 발생한다"임을 알 수 있다. 따라서 x에 들어갈 말로 가장 적절한 것은 ②이다.
```

**패턴 C — 비유/메타포 빈칸:**
```
[PK POINT]
빈칸 x를 무한한 경제 확장의 비유로 타게팅합니다. 빈칸 문장을 읽으면 '무한한 경제 확장을 추구하는 것은 x를 반영한다'가 되므로, x가 어떤 비유인지를 찾으며 글을 읽습니다. #autoimmune condition, #living Earth에 태깅하며 글을 읽으면 이 지문의 god는 "무한 성장 추구는 암세포의 논리와 같다"임을 알 수 있다. 따라서 x에 들어갈 말로 가장 적절한 것은 ③이다.
```

### 필수 워딩
- "빈칸 x를 ~로 타게팅합니다." (첫 문장, 반드시 이 형태)
- "빈칸 문장을 읽으면 '~'가 되므로" (빈칸 해석)
- "태깅하며 글을 읽으면" (태깅 분석)
- "이 지문의 god는 ~임을 알 수 있다" (god 파악)
- "x에 들어갈 말로" (결론)

## 톤 & 말투 규칙

1. **강의체**: ~입니다/~됩니다/~합니다 (존댓말)
2. **직접 지시**: "~를 태깅해 준다", "~에 주목하여 풀어야 합니다"
3. **학생 시점 유도**: "~라는 것을 알 수 있으므로", "~임을 파악해야 합니다"
4. **자신감 있는 단정**: "답이 된다", "핵심입니다", "주목해야 합니다"
