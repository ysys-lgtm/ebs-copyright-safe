# ebs-copyright-safe

EBS 교재에서 출처 없는 문제를 식별하고, **3중 저작권 안전 처리**를 적용하여 저작권 접촉 위험이 없는 문제로 변환한다.

## 핵심 원칙: 3중 저작권 보호

| 단계 | 처리 내용 | 목적 |
|------|----------|------|
| 1단계 | **본문 paraphrase** | 표현의 유사성 제거 |
| 2단계 | **유형 변경** | 구조적 유사성 제거 (풀이 논리 자체가 달라짐) |
| 3단계 | **선택지 재작성** | 문제 구성 요소의 유사성 제거 |

> **핵심**: 같은 지문 + 같은 유형 = 동일한 풀이 논리 = 저작권법상 "실질적 유사성" 위험.
> 유형을 반드시 변경해야 구조적 유사성까지 제거된다.

## 출처 구분 기준

### 출처 있음 → 수정 제외 (그대로 유지)
- "사관학교", "평가원", "수능", "학력평가", "교육청" 등 출처 표시가 있는 문제
- 이 문제들은 공공 기출이므로 저작권 문제 없음

### 출처 없음 → EBS 문제 → 전면 수정 필요
- 문제 번호 위/옆에 아무 출처도 없는 문제
- EBS 교재 자체 문제로 저작권 접촉 위험

## 입력 파일 형식

### HWP 파일 (한글 문서)
- `pyhwp` 패키지로 텍스트 추출 (`pip3 install pyhwp`)
- 추출 명령: `hwp5txt 파일명.hwp > 출력.txt`
- HWP는 바이너리 형식이므로 직접 수정 불가 → .txt로 결과 출력

### 텍스트 파일
- 이미 추출된 텍스트 파일도 입력 가능
- `db-text` 스킬 출력 형식도 호환

## 유형 변환 규칙

### 원칙
1. **원본과 동일한 유형은 절대 사용 금지**
2. 지문 내용에 적합한 유형 선택
3. 같은 파일 내 유형 다양성 확보

### 유형 변환 가이드

| 원본 유형 | → 변환 가능 유형 (우선순위) |
|-----------|--------------------------|
| 주제 | → 제목, 빈칸, 요약 |
| 요지 | → 제목, 주제, 빈칸 |
| 주장 | → 주제, 제목, 빈칸 |
| 제목 | → 주제, 빈칸, 요약 |
| 빈칸 | → 주제, 제목, 밑줄(함의) |
| 순서(A/B/C) | → 빈칸, 주제, 삽입 |
| 삽입 | → 빈칸, 주제, 순서 |
| 무관한 문장 | → 주제, 제목, 빈칸 |
| 어법 | → 어휘, 빈칸 |
| 어휘 | → 어법, 빈칸 |
| 요약 | → 주제, 제목, 빈칸 |
| 목적 | → 주제, 빈칸 |

## 본문 paraphrase 규칙

### 모든 유형 공통
- **핵심 논지/구조 유지**: 글의 주제와 논리 전개 방향은 동일
- **표현 전면 변경**: 단어 교체, 문장 구조 변경, 예시 교체
- **원본과 표면적으로 다르게**: 같은 문장이 하나도 없어야 함

### 특수 유형별 추가 규칙

| 유형 | paraphrase 시 주의사항 |
|------|----------------------|
| 순서 → 다른 유형 | (A)(B)(C) 단락을 정답 순서대로 합쳐 하나의 연속 지문으로 |
| 삽입 → 다른 유형 | 삽입 문장을 정답 위치에 넣어 완전한 지문으로 통합 |
| 무관한 문장 → 다른 유형 | 무관한 문장(정답)을 제거한 뒤 남은 지문 사용 |
| 어법/어휘 → 다른 유형 | 밑줄/번호 표시 제거, 정상 문장으로 복원 후 사용 |

### 구조적 유사성 금지 사항

다음과 같은 수정은 **저작권 위험이 여전히 남아 부적절**하다:

| 부적절한 수정 | 위험한 이유 |
|-------------|-----------|
| 이름만 교체 (Edison→Swan) | 서사 구조 동일 |
| 비율만 변경 (5%/95% → 10%/90%) | 논리 프레임 동일 |
| 예시만 교체 (신발→음료) | 일화 구조 동일 |
| 유형 유지 + 선택지만 변경 | 풀이 논리 동일 |

> **올바른 수정**: 소재만 유지하고, 관점/맥락/유형을 모두 바꿔야 한다.
> 예: Edison 전구 발명(시행착오 서사) → 전기 조명의 사회적 영향(분석적 서술) + 유형 변경

## 유형별 문제 생성 규칙

### 빈칸으로 변환
- 지문에서 **핵심 표현** 하나를 `________________________________`으로 처리
- 빈칸 위치: 글의 주제문 또는 결론부의 핵심 어구
- 영어 선택지 5개 작성
- 정답: 빈칸에 들어갈 가장 적절한 표현
- 오답: 그럴듯하지만 문맥에 맞지 않는 표현

### 주제로 변환
- 질문: "다음 글의 주제로 가장 적절한 것은?"
- **선택지 반드시 영어** 5개 (한국어 혼용 금지)
- 형식: `the + [명사구]` 또는 `how/why/what + 절`
- 정답: 글의 핵심 논지를 정확히 담은 선택지
- 오답: 부분적으로만 맞거나, 과도하게 일반화하거나, 세부 사항만 언급

### 제목으로 변환
- 질문: "다음 글의 제목으로 가장 적절한 것은?"
- 영어 선택지 5개 (함축적이고 흥미로운 표현)
- 정답: 핵심 주제를 포괄하는 제목
- 오답: 너무 좁거나 넓거나, 지엽적 내용에 초점

### 요약으로 변환
- 지문 핵심을 1~2문장으로 요약, 핵심 키워드 2개를 (A)(B) 빈칸 처리
- 선택지 5개 쌍: (A) + (B) 조합
- 정답: 핵심 내용을 정확히 반영하는 쌍
- 오답: 한쪽만 맞거나 둘 다 미묘하게 부정확

### 어휘로 변환 (어법 → 어휘)
- 본문의 5개 핵심 단어에 ①②③④⑤ 밑줄 표시
- 1개를 문맥상 **부적절한 단어**로 교체 (의미 반대 또는 문맥 위반)
- 나머지 4개는 원문 그대로 유지
- 질문: "밑줄 친 단어 중, 문맥상 낱말의 쓰임이 적절하지 않은 것은?"

### 삽입으로 변환
- 지문에서 논리적으로 이동 가능한 문장 1개 추출
- 추출된 문장을 "주어진 문장"으로 제시
- 본문에 ①~⑤ 위치 표시 (추출된 문장 자리 포함)
- 정답: 논리적 흐름상 가장 자연스러운 위치

## 워크플로우

### Step 1: 정보 수집

```
AskUserQuestion: "변형할 파일 경로를 알려주세요. (HWP 또는 텍스트 파일)"
```

- HWP인 경우 `pyhwp`로 텍스트 추출 진행
- 여러 파일 동시 처리 가능

### Step 2: 출처 분류

각 문제를 분류:
- **출처 있음** (사관학교/평가원/학력평가/교육청/수능) → 목록만 기록, 수정 제외
- **출처 없음** (EBS) → 수정 대상으로 분류

### Step 3: 본문 paraphrase (1단계)

출처 없는 모든 문제의 본문을 paraphrase:
- 핵심 논지/구조 유지, 표현 전면 변경
- 순서/삽입/무관한 문장 등 특수 유형은 지문 통합 후 paraphrase

### Step 4: 유형 변경 (2단계)

원본 유형과 **반드시 다른 유형**으로 변환:
- 유형 변환 가이드 참고
- 같은 파일 내 유형 다양성 확보 (빈칸/주제/제목/요약 고르게 분배)

### Step 5: 선택지 재작성 (3단계)

새로운 유형에 맞는 선택지 5개 작성:
- 정답은 지문 내용을 정확히 반영
- 오답은 그럴듯하지만 수능 수준의 변별력 유지

### Step 6: 결과 저장

두 개의 파일을 생성:

**파일 1: 수정 문제 본문** (`{폴더명}_수정문제_최종.txt`)
```
================================================================
[주차/강 번호번] 원본: X → 변경: Y
================================================================
정답: ①

(문제 지시문)
(paraphrase된 지문)

① ...
② ... ← 정답
③ ...
④ ...
⑤ ...

[원본 메모]
- 원본 유형: ...
- 변경 이유: ...

---
```

**파일 2: 해석 및 해설** (`{폴더명}_해석해설.txt`)
```
[번호] 원본: X → 변경: Y
정답: ①

[해석]
(한국어 전체 번역)

[해설]
- 핵심 논지: ...
- 정답 근거: ...
- 오답 분석: ②... ③... ④... ⑤...

---
```

### Step 7: 최종 검수 (누락 감사)

**반드시 실행**: 결과 파일 저장 후, 다음 검수를 수행한다.

1. **전수 대조**: 최종 파일에서 실제 포함된 문제 번호 목록을 추출한다.
2. **누락 탐지**: 원본 파일의 전체 문제 번호(1~30, Top, 사관 등)와 대조하여, 출처 있는 문제도 아니고 변형 문제에도 없는 "빠진 번호"가 있는지 확인한다.
3. **검증 공식**: `출처 있는 문제 수 + 변형 완료 문제 수 = 원본 전체 문제 수` 가 반드시 성립해야 한다.
4. **누락 발견 시**: 즉시 해당 문제의 원본을 추출하여 3중 저작권 보호 처리를 적용하고, 결과 파일(본문 + 해석해설)에 정확한 위치에 삽입한다.
5. **"이전 작업에서 변경됨" 판단 금지**: 최종 파일에 실체(지문+선지)가 포함되지 않은 문제는 "완료"로 간주하지 않는다. 반드시 실체가 파일에 존재해야 완료이다.

```
검수 결과 예시:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[32주차 검수]
전체: 30문제 | 출처있음: 7개 | 변형완료: 23개 | 누락: 0개 ✓
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 8: 결과 안내

- 저장된 파일 경로 목록
- 출처별 분류 요약 (수정 대상 vs 제외)
- 유형 변환 내역 표
- 전체 문제 수 통계
- **검수 결과** (주차별 누락 여부 표)

## 주의사항

- **정답 번호 유지 불필요**: 유형이 바뀌므로 새로운 정답 번호로 설정 가능
- **구조적 유사성 금지**: 이름/숫자만 바꾸는 수준의 수정은 저작권 위험 잔존
- **소재는 유지 가능**: 같은 주제(예: 전기 조명)라도 관점/맥락을 바꾸면 안전
- **주제 추론 선지**: 반드시 영어로 작성 (한국어 혼용 금지)
- **병렬 처리 권장**: 주차/강별로 Agent를 병렬 실행하여 처리 속도 향상
- HWP 파일은 바이너리이므로 결과는 .txt로 출력 (한글 앱에서 복사-붙여넣기 안내)
- **"이전 작업에서 변경됨" 판단 금지**: 최종 파일에 문제 실체(지문+선지)가 없으면 미완료. 실체 없이 "변경됨"으로 기록하고 건너뛰는 것은 누락으로 간주
- **최종 검수 필수**: Step 7의 전수 대조를 반드시 수행. `출처 있음 + 변형 완료 = 전체 문제 수` 공식이 성립하지 않으면 작업 미완료

## 연계 스킬

- **db-text**: 수능 교재 PDF에서 지문 추출
- **suneung-transform**: EBS 지문의 유형 변형 (원문 유지 방식) — 본 스킬은 저작권 안전까지 보장하는 상위 스킬
- **suneung-pipeline**: PDF → 추출 → 변형 원클릭 파이프라인
